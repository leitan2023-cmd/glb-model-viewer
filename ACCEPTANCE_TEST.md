# MVP 验收脚本

## 环境准备

- 浏览器：Chrome/Chromium 90+
- 测试模型：准备 2-3 个不同的 GLB 文件
- 网络：正常连接

---

## G. 验收脚本

### G1. 模型管理验收

#### G1.1 上传模型不刷新页面
```
步骤：
1. 打开应用
2. 点击"上传模型"按钮
3. 选择第一个 GLB 文件上传
4. 等待模型加载完成
5. 点击"上传模型"按钮
6. 选择第二个 GLB 文件上传
7. 再次上传第三个 GLB 文件

验收标准：
✓ 页面不刷新
✓ 每次上传后模型正确替换
✓ 统计信息（mesh 数、三角面数）正确更新
✓ 树结构正确更新
✓ 选择状态清空
✓ 相机自动聚焦到新模型
```

#### G1.2 历史记录持久化
```
步骤：
1. 完成 G1.1 的上传（3 个模型）
2. 点击左侧"历史模型"标签页
3. 观察历史列表

验收标准：
✓ 显示 3 条历史记录
✓ 每条记录包含：缩略图、模型名、上传时间
✓ 显示统计信息（mesh 数、三角面数）
✓ 缩略图清晰可见
```

#### G1.3 历史模型快速切换
```
步骤：
1. 在历史模型列表中点击第一条记录的"加载"按钮
2. 等待模型加载
3. 观察模型是否正确加载

验收标准：
✓ 模型快速加载（< 1 秒）
✓ 模型内容正确
✓ 统计信息正确
✓ 页面不刷新
```

#### G1.4 删除历史记录
```
步骤：
1. 在历史模型列表中点击某条记录的"删除"按钮
2. 观察列表变化

验收标准：
✓ 记录被删除
✓ 列表更新
✓ 统计数字更新
```

#### G1.5 清空历史
```
步骤：
1. 点击"清空历史"按钮
2. 确认操作

验收标准：
✓ 所有历史记录被删除
✓ 列表显示"暂无历史记录"
```

#### G1.6 历史记录刷新后仍存在
```
步骤：
1. 上传一个模型
2. 点击左侧"历史模型"标签页，确认记录存在
3. 按 F5 刷新页面
4. 等待页面加载完成
5. 点击左侧"历史模型"标签页

验收标准：
✓ 刷新后历史记录仍然存在
✓ 缩略图仍然显示
✓ 统计信息正确
```

#### G1.7 拖拽上传
```
步骤：
1. 打开应用（未加载模型）
2. 将 GLB 文件拖拽到中央区域
3. 释放鼠标

验收标准：
✓ 模型加载成功
✓ 统计信息正确
✓ 历史记录保存
```

### G2. 事件记录验收

#### G2.1 运维模式启用/禁用
```
步骤：
1. 加载一个模型
2. 点击顶部"🔧 运维模式"按钮
3. 观察按钮状态

验收标准：
✓ 按钮变为高亮（默认样式变为 default）
✓ 按钮保持可点击
✓ 可以再次点击禁用
```

#### G2.2 创建事件
```
步骤：
1. 启用运维模式
2. 在模型上点击一个构件
3. 在弹窗中填写信息：
   - 事件类型：选择"搭载定位"
   - 标题：输入"测试事件"
   - 备注：输入"这是一个测试"
4. 点击"保存"

验收标准：
✓ 事件编辑弹窗打开
✓ 目标构件名称自动填充
✓ 已记录点数显示为 1
✓ 事件保存成功
✓ 弹窗关闭
✓ 显示"事件已保存"提示
```

#### G2.3 事件列表显示
```
步骤：
1. 完成 G2.2 的事件创建
2. 点击左侧"事件记录"标签页

验收标准：
✓ 显示刚创建的事件
✓ 显示事件标题、类型、构件名
✓ 显示创建时间
✓ 显示操作按钮（👁️ ✏️ 🗑️）
```

#### G2.4 事件搜索
```
步骤：
1. 创建 2-3 个不同的事件
2. 在事件列表的搜索框中输入事件标题的部分内容
3. 观察列表变化

验收标准：
✓ 列表只显示匹配的事件
✓ 搜索实时响应
✓ 清空搜索框后显示全部事件
```

#### G2.5 事件筛选
```
步骤：
1. 创建不同类型的事件（至少 2 种）
2. 点击"全部类型"下拉框
3. 选择某个类型

验收标准：
✓ 列表只显示选中类型的事件
✓ 统计数字更新
✓ 选择"全部类型"后显示全部事件
```

#### G2.6 事件定位
```
步骤：
1. 在事件列表中点击某个事件
2. 点击事件的"👁️"按钮

验收标准：
✓ 模型自动聚焦到事件所在构件
✓ 构件高亮显示
✓ 相机平滑移动到构件
```

#### G2.7 事件编辑
```
步骤：
1. 在事件列表中点击某个事件的"✏️"按钮
2. 修改标题和备注
3. 点击"保存"

验收标准：
✓ 编辑弹窗打开，显示原数据
✓ 修改保存成功
✓ 列表显示更新后的数据
✓ 显示"事件已保存"提示
```

#### G2.8 事件删除
```
步骤：
1. 在事件列表中点击某个事件的"🗑️"按钮
2. 在确认对话框中点击"确定"

验收标准：
✓ 事件被删除
✓ 列表更新
✓ 统计数字更新
```

#### G2.9 事件类型特定字段
```
步骤：
1. 创建"立柱"类型的事件
2. 在编辑弹窗中填写：
   - 高度：10.5
   - 垂直度：5.2
3. 保存

验收标准：
✓ 弹窗显示"高度"和"垂直度"字段
✓ 字段值正确保存
✓ 编辑时字段值正确显示
```

#### G2.10 事件数据持久化
```
步骤：
1. 创建几个事件
2. 刷新页面（F5）
3. 等待页面加载完成
4. 点击"事件记录"标签页

验收标准：
✓ 事件列表仍然显示
✓ 事件数据完整
✓ 可以继续编辑/删除事件
```

### G4. 量测工具验收

#### G4.1 量测模式启用
```
步骤：
1. 加载一个模型
2. 点击顶部"📏 量测"按钮

验收标准：
✓ 按钮变为高亮
✓ 可以点击模型
```

#### G4.2 测量距离
```
步骤：
1. 启用量测模式
2. 在模型上点击第一个点
3. 在模型上点击第二个点
4. 观察结果

验收标准：
✓ 显示红色线段
✓ 两端显示红色小球
✓ 中点显示距离标签
✓ 距离值正确（精确到 2 位小数）
✓ 量测模式自动禁用
✓ 显示"测量完成"提示
```

#### G4.3 多次量测
```
步骤：
1. 完成一次量测
2. 再次启用量测模式
3. 进行第二次量测

验收标准：
✓ 第一次量测的线段仍然显示
✓ 第二次量测的线段也显示
✓ 两次量测不互相影响
```

### G5. 数据联动验收

#### G5.1 StableKey 生成
```
步骤：
1. 加载一个模型
2. 打开浏览器 DevTools
3. 在 Console 中执行：
   db.transaction(['events']).objectStore('events').getAll()

验收标准：
✓ 事件中的 targetKey 不为空
✓ targetKey 格式一致
```

#### G5.2 模型替换后数据仍有效
```
步骤：
1. 加载模型 A，创建事件
2. 上传模型 B
3. 再上传模型 A

验收标准：
✓ 模型 A 的事件仍然存在
✓ 标注仍然显示在正确位置
```
