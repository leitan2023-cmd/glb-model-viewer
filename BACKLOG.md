# MVP 未实现功能清单

## P1 功能（高优先级）

### P1.1 导入状态 JSON
**优先级**：高  
**工作量**：中（2-3 天）  
**描述**：支持导入构件状态数据文件，自动按 stage/status 着色模型

**需求**：
- 添加"导入状态"按钮到工具栏
- 支持 JSON 格式的状态文件
- 文件格式：
  ```json
  {
    "parts": {
      "stableKey1": {
        "stage": "completed",
        "status": "passed",
        "metrics": { "quality": 95 }
      },
      "stableKey2": {
        "stage": "in_progress",
        "status": "pending"
      }
    }
  }
  ```
- 导入后自动应用状态着色
- 支持预览导入结果

**实现步骤**：
1. 创建状态导入对话框组件
2. 实现 JSON 文件解析
3. 实现状态验证
4. 集成到 Home.tsx
5. 测试和优化

### P1.4 状态历史记录
**优先级**：中  
**工作量**：中（2-3 天）  
**描述**：记录构件状态变化历史，支持时间轴回放

**需求**：
- 记录每次状态变化
- 显示状态变化时间线
- 支持按时间查询历史状态
- 支持时间轴回放

**数据结构**：
```typescript
{
  modelId: string;
  stableKey: string;
  changes: Array<{
    timestamp: number;
    oldState: PartState;
    newState: PartState;
    changedBy: string;
  }>;
}
```

**实现步骤**：
1. 修改 StateStore 添加历史记录功能
2. 创建时间线组件
3. 实现回放逻辑
4. 测试和优化

## 贡献指南

### 如何添加新功能

1. **创建 Issue**
   - 描述功能需求
   - 提供用例
   - 估计工作量

2. **设计阶段**
   - 讨论技术方案
   - 评审设计文档
   - 获得批准

3. **开发阶段**
   - 创建特性分支
   - 编写代码和测试
   - 提交 Pull Request

4. **审查阶段**
   - 代码审查
   - 测试审查
   - 文档审查

5. **发布阶段**
   - 合并到 main 分支
   - 发布新版本
   - 更新文档
