# GLB Model Viewer - MVP 功能说明

## 已实现功能

### 1. 模型管理（Model Management）

#### 1.1 上传模型（不刷新页面）
- **操作**：点击顶部工具栏的"上传模型"按钮
- **功能**：
  - 打开文件选择器（支持 .glb, .gltf）
  - 上传后自动替换当前模型
  - 自动释放旧模型资源（geometry/material/texture）
  - 重新计算模型统计信息（mesh 数、三角面数）
  - 自动保存到历史记录

#### 1.2 历史记录（IndexedDB 持久化）
- **位置**：左侧面板"历史模型"标签页
- **功能**：
  - 显示最近 20 条模型记录
  - 每条记录包含：缩略图、模型名、上传时间、统计信息
  - 点击"加载"按钮快速切换模型
  - 点击"删除"按钮删除单条记录
  - "清空历史"按钮清空所有记录
  - 刷新页面后历史记录仍然存在

#### 1.3 拖拽上传
- **操作**：将 GLB 文件拖拽到中央区域
- **功能**：同"上传模型"按钮功能

---

### 2. 事件记录系统（Event Recording）

#### 2.1 运维模式
- **启用**：点击顶部工具栏的"🔧 运维模式"按钮
- **功能**：
  - 在运维模式下，点击模型上的任何构件会打开事件编辑弹窗
  - 支持创建 4 种事件类型：
    - **切割线**（cut_line）：代码
    - **搭载定位**（bearing）：位置
    - **立柱**（column）：高度、垂直度
    - **背烧**（backburn）：面积、偏移、原因

#### 2.2 事件编辑
- **弹窗字段**：
  - 事件类型（下拉选择）
  - 标题（文本输入）
  - 目标构件（自动填充，不可编辑）
  - 备注（多行文本）
  - 类型特定字段（根据事件类型显示）
  - 已记录的点数显示

#### 2.3 事件列表
- **位置**：左侧面板"事件记录"标签页
- **功能**：
  - 显示当前模型的所有事件
  - 搜索功能：按标题、备注、构件名搜索
  - 筛选功能：按事件类型筛选
  - 操作按钮：
    - 👁️ 定位：跳转到事件所在构件
    - ✏️ 编辑：编辑事件
    - 🗑️ 删除：删除事件
  - 显示统计信息（总数、显示数）

#### 2.4 事件持久化
- **存储**：IndexedDB（按 modelId 分桶）
- **数据结构**：
  ```typescript
  {
    id: string;                    // 唯一标识
    modelId: string;               // 所属模型
    type: EventType;               // 事件类型
    title: string;                 // 标题
    targetKey: string;             // 构件 stableKey
    targetName: string;            // 构件名称
    points: Array<{x, y, z}>;      // 3D 点坐标
    fields: Record<string, any>;   // 类型特定字段
    note: string;                  // 备注
    attachments: string[];         // 附件 URL
    createdAt: number;             // 创建时间
    updatedAt: number;             // 更新时间
  }
  ```

### 3. 数据联动（Data Integration）

#### 3.1 StableKey 机制
- **概念**：为每个构件生成稳定的唯一标识
- **用途**：
  - 绑定事件到构件
  - 绑定状态数据到构件
  - 支持模型更新后数据仍然有效

#### 3.2 状态数据（State Data）
- **存储**：IndexedDB（按 modelId）
- **数据结构**：
  ```typescript
  {
    modelId: string;
    parts: {
      [stableKey]: {
        stage: string;           // 施工阶段
        status: string;          // 状态
        metrics: Record<string, any>; // 其他指标
        updatedAt: number;
      }
    }
  }
  ```

#### 3.3 导入状态 JSON（P1 功能）
- **计划**：支持导入状态 JSON 文件
- **功能**：
  - 按 stage/status 自动着色模型
  - 更新构件状态数据

---

### 4. 标注可视化（Annotation Visualization）

#### 4.1 点标注
- **显示**：蓝色小球 + 标签
- **用途**：标记单个位置

#### 4.2 线标注
- **显示**：橙色线段 + 端点球 + 距离标签
- **用途**：标记两点间的距离和位置

#### 4.3 标注管理
- **自动创建**：事件创建时自动生成标注
- **自动删除**：事件删除时自动删除标注
- **高亮**：选中事件时高亮对应标注

---

### 5. 量测工具（Measurement Tool）

#### 5.1 启用量测模式
- **操作**：点击顶部工具栏的"📏 量测"按钮
- **功能**：
  - 点击第一个点
  - 点击第二个点
  - 自动计算距离并显示
  - 显示红色线段和距离标签

#### 5.2 量测结果
- **显示**：
  - 两个端点（红色小球）
  - 连接线（红色）
  - 距离标签（中点上方）
- **单位**：米（m）

---

### 6. 已有能力保持

#### 6.1 树/模型互选
- 在结构树中选择构件，模型中高亮显示
- 在模型中点击构件，结构树中自动展开并选中

#### 6.2 聚焦功能
- 双击构件自动聚焦
- "重置视角"按钮恢复默认视角
- "适配模型"按钮自动调整相机以显示整个模型

#### 6.3 高亮效果
- 选中构件时，其他构件变为 60% 透明
- 支持多选（Ctrl+点击）

#### 6.4 相机控制
- 鼠标拖拽旋转
- 滚轮缩放
- 右键平移

#### 6.5 Debug HUD
- 显示拾取信息（pickables 数、hits 数等）
- 可通过"Advanced"面板开关

## 数据存储

### IndexedDB 数据库结构

```
Database: glb-model-viewer-db

Stores:
  1. model-history
     - keyPath: id
     - 存储模型文件和缩略图

  2. model-state
     - keyPath: modelId
     - 存储构件状态数据

  3. events
     - keyPath: id
     - indexes: modelId, targetKey
     - 存储事件记录
```

## 使用示例

### 1. 创建事件
```
1. 点击"🔧 运维模式"启用运维模式
2. 在模型上点击要标注的构件
3. 在弹窗中填写事件信息
4. 点击"保存"
```

### 2. 量测距离
```
1. 点击"📏 量测"启用量测模式
2. 点击第一个点
3. 点击第二个点
4. 自动显示距离
```

### 3. 查看历史模型
```
1. 点击左侧"历史模型"标签页
2. 点击某条记录的"加载"按钮
3. 模型自动加载
```

## 故障排除

### 历史记录不显示
- 检查浏览器是否支持 IndexedDB
- 检查浏览器隐私设置
- 清除浏览器缓存后重试

### 标注不显示
- 检查事件是否正确保存
- 检查构件 stableKey 是否正确生成
- 打开 Debug HUD 查看详细信息

### 量测工具不工作
- 确保模型已加载
- 确保量测模式已启用
- 尝试重新启用量测模式
